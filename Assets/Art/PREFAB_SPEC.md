# 임시/최종 아트 프리팹 규격서 (로드맵 3·7)

> [`MapLoader`](../Scripts/Level/MapLoader.cs)의 **프리팹 슬롯**에 넣을 시각 프리팹을 만들 때의 사양.
> seam은 코드로 완성돼 있음(로드맵 2). **이 규격대로 프리팹만 만들어 슬롯에 드래그하면 끝** — 게임 코드 변경 0.
> 슬롯이 비어 있으면 지금처럼 색 사각형으로 폴백되므로, 만든 것부터 하나씩 채워도 됨.

## 공통 규칙 (모든 프리팹)

- **구성**: 루트 GameObject + `SpriteRenderer`(필수) + 선택적 `Animator`. **그게 전부.**
- **넣지 말 것**: Collider2D · Rigidbody2D · 게임 로직 스크립트(Mirror/Prism/GateDetector/PlayerController 등).
  충돌·물리·광학은 코드가 루트에 만드는 **실제 오브젝트**가 담당하고, 프리팹은 그 오브젝트의 `"visual"` 자식으로만 들어간다.
- **Pixels Per Unit(PPU) = 40** 으로 임포트(맵 규약 `cell_px=40`, 1셀=1유닛). → 텍스처 40px = 1유닛.
- **Pivot = Center(0.5, 0.5)**. 코드가 시각을 오브젝트 중심(localPosition 0)에 놓기 때문.
- **크기 = 최종 크기로 저작**(아래 표). 플레이스홀더의 얇은 막대/작은 사각형 스케일은 프리팹에 **적용되지 않는다**(거울·사다리 예외는 아래).
- **sortingOrder**: 코드가 `표의 base + 프리팹 내부 상대순서`로 세팅. 프리팹 안에서 여러 SpriteRenderer를 쓰면 상대 순서(0,1,2…)만 맞춰두면 됨.
- **애니메이션**: 코드는 Animator 파라미터를 건드리지 않는다. 임시 단계에선 **정적 스프라이트** 또는 **스스로 루프하는 idle 애니메이션**만 사용(거울 회전·게이트 개폐는 코드가 transform·색으로 처리).

## 제작 단위 (중요)

- **그림 그릴 때 쓰는 단위 = 픽셀(px). `1칸(1유닛) = 40px`.**
- 아래 표의 **`제작 px`가 실제 캔버스 크기**다. 그걸 **PPU 40**으로 임포트하면 표의 `월드(unit)` 크기로 배치된다.
- 예) 40×40px → 1×1유닛(한 칸), 24×36px → 0.6×0.9유닛(플레이어).

### 부속(2조각) 오브젝트 — 아트가 나뉘는 것

- **랜즈 = `lensPrefab`(빛나는 부분) + `torchPrefab`(장착 횃불)**. 둘 다 **랜즈 위치에 중심 정렬**되고, 횃불은 랜즈 **뒤(order 4)**에 회전 없이 깔린다. 횃불 몸통을 아래로 뻗게 하려면 세로로 긴 캔버스에서 **랜즈 장착부가 위쪽**에 오게 그릴 것.
- **거울 = `mirrorPrefab`(돌아가는 반사면) + `mirrorMountPrefab`(고정 거치대)**. 거치대는 **거울 뒤(order 4)**에 **회전하지 않고** 고정, 반사면만 코드가 `−angle`로 돌린다. 거치대 아트는 거울 중심에 정렬되므로, 잡아주는 팔/받침을 캔버스 안에서 원하는 위치에 그리면 된다. (고정 거울도 같은 거치대 사용)
- **게이트 = `gatePrefab`(수광부, 별도 오브젝트) + `gateDoorPrefab`(문)**. 수광부와 문은 **서로 다른 위치의 별개 오브젝트**다(수광부=맵의 `gate.pos`, 문=`gate_open_zone`).
  - **문 = 긴 하나의 아트**. 개폐존(전 스테이지 공통 **3칸**)을 덮는 단일 오브젝트로 생성된다. **세로로 긴 문 하나(1×3 = 40×120px)만 그리면** 코드가 존에 맞춰 넣는다.
  - **바닥 해치(가로형 존)는 코드가 같은 아트를 90° 눕혀** 3×1에 맞춘다 → **1×3짜리 하나로 세로문 3개 + 바닥해치 1개 전부 왜곡 없이 커버**. 그래서 문 아트는 **위/아래 구분이 크지 않은(90° 돌려도 어색하지 않은)** 디자인이 안전.
  - 코드가 원본 크기를 존에 맞추므로 정확히 1×3이 아니어도 되지만(자동 스케일), **1×3(40×120)으로 그리면 늘어남 없이 딱 맞음**.

## 주의가 필요한 3가지 (seam 계약)

1. **거울(반사면 `mirrorPrefab`)** — 코드가 `−angle` 로 회전시킨다(반사면 정렬). **각도 0°(수평) 상태로 저작**하고 회전은 코드에 맡길 것. 거치대(`mirrorMountPrefab`)는 **회전하지 않는다**. 프리즘의 45°는 플레이스홀더 전용이라 프리즘 프리팹은 **정립**으로.
2. **사다리** — 높이 `h`가 데이터 종속 → 코드가 프리팹에 **세로 스케일 ×h**를 건다. 1칸(1유닛) 높이로 저작하고 **세로 타일/9-slice로 늘어나도 자연스럽게** 그릴 것(단순 스프라이트는 h배로 늘어나 보임). 가로는 0.6유닛 유지(스케일 x=1).
3. **게이트/문 색 피드백** — 열림/닫힘 시 코드가 **`visual`의 첫 SpriteRenderer 색을 바꾼다**(수광부=초록, 문=반투명). 색이 제대로 읽히려면 대표 스프라이트를 **중립/흰색 베이스**로 두는 게 안전(틴트가 곱해짐). 색 대신 애니메이터/자식으로 상태를 표현하고 싶으면 그래도 됨 — 그 경우 색 틴트는 무시된다(코드 null-safe).

- **랜즈 방향**: `lensPrefab`을 넣으면 플레이스홀더 방향 표시 점은 자동 생략된다. **단 코드는 랜즈를 빔 방향으로 회전시키지 않으므로**(현재), 방향 비의존 **대칭 아트**를 권장. 방향성 아트가 꼭 필요하면 랜즈에 회전을 주입하는 소규모 코드 변경이 필요(요청 시 추가).
- **스폰 표식**: 최종 빌드에선 대개 불필요 → `spawnPrefab`은 비워두면 표식이 사라진다(색 사각형도 안 나옴은 아님 — 비우면 흰 사각형 폴백이 남으니, 표식을 없애려면 투명 1px 스프라이트 프리팹을 넣거나 별도 처리).

## Unity에서 만들고 배정하는 순서

1. 스프라이트 PNG를 `Assets/Art/Sprites/` 에 넣고, 각 텍스처를 선택 → Inspector에서 **Sprite (2D and UI)**, **Pixels Per Unit = 40**, **Pivot = Center**, (픽셀아트면) **Filter Mode = Point / Compression = None**.
2. 빈 GameObject에 `SpriteRenderer` 추가 → 스프라이트 지정 → `Assets/Art/Prefabs/` 로 드래그해 프리팹화. (콜라이더·스크립트 넣지 말 것)
3. **SampleScene**에서 `MapLoader` 컴포넌트 선택 → **프리팹 슬롯**에 해당 프리팹을 드래그.
4. ▶ Play(또는 컴포넌트 우클릭 Build)로 확인. 슬롯이 빈 것은 색 사각형으로 남는다.
5. 만족하면 **씬 저장**(슬롯은 씬의 MapLoader에 직렬화됨) + 스프라이트/프리팹/씬을 커밋.

> ⚠️ 새로 추가한 애셋의 `.meta`는 Unity가 임포트 시 생성한다 → 반드시 **에디터로 한 번 연 뒤** `.meta`까지 함께 커밋할 것(안 그러면 다른 PC에서 참조가 깨짐).
